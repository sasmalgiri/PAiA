Getting started

  1. Install Ollama from ollama.com/download and start it.
  2. Pull a model: ollama pull llama3.2
  3. Download PAiA from the download page.
  4. Run the installer. First launch shows a 3-step welcome wizard.
  5. Click the ball in your screen corner. Type a message or hit the mic.

Knowledge collections (RAG)

RAG lets PAiA answer questions using documents you provide. To set it up:

  1. Pull an embedding model: ollama pull nomic-embed-text
  2. Open Settings → Knowledge → Create collection
  3. Drop in PDFs, Markdown, JSON, CSV, code, or any text file
  4. Wait for chunking + embedding (a few seconds per document)
  5. In the chat panel, click the 📚 button to attach the collection to a thread
  6. Ask questions — PAiA retrieves the top-5 chunks and cites them as [1], [2]…

Vision models

PAiA supports any vision model Ollama supports. To use:

  1. Pull a vision model: ollama pull llava (or bakllava, moondream, llama3.2-vision, etc.)
  2. Switch to the vision model in the chat panel header dropdown
  3. Drag an image into the composer, paste from clipboard with Ctrl+V, or use /image
  4. Ask your question

If you attach an image while a non-vision model is selected, PAiA shows a yellow warning banner so you don't waste a roundtrip.

MCP tool servers

PAiA is a Model Context Protocol client. Any MCP server you can run as a child process can be added under Settings → Tools.

Example: the official filesystem MCP server gives PAiA read/write access to a folder you choose.

Settings → Tools → Add MCP server
  Name:    filesystem
  Command: npx
  Args:    -y @modelcontextprotocol/server-filesystem /path/to/folder

By default, every tool call requires explicit approval through a modal. You can whitelist trusted tools per-server under "Auto-approve tools".

Cloud models (opt-in)

PAiA's privacy guarantees only hold for local models. To use OpenAI / Anthropic / OpenAI-compatible providers:

  1. Settings → General → Allow cloud models (turn on)
  2. Settings → Models → Cloud providers → enable a provider → paste your API key
  3. Pick the cloud model from the chat panel header dropdown (cloud models are prefixed with ☁)

Activating a license

  1. Buy Pro from the pricing page
  2. You'll receive an email with a JSON license blob
  3. Settings → License → Activate
  4. Paste the entire JSON block

Verification is offline — Ed25519 signature checks happen on your machine. No phone-home.

FAQ

Where is my data?

In your OS user data folder. On Windows: %APPDATA%/paia. On macOS: ~/Library/Application Support/paia. On Linux: ~/.config/paia.

Can I export my chat history?

The database is plain SQLite. You can copy paia.sqlite from your user data folder and open it with any SQLite tool.

How do I reset everything?

Quit PAiA, delete the user data folder, relaunch. You'll see the onboarding wizard again.

What models work best?

For chat: llama3.2, qwen2.5, mistral. For RAG embeddings: nomic-embed-text. For vision: llava, bakllava, moondream. For code: deepseek-coder-v2, qwen2.5-coder.

How much disk space does PAiA itself use?

The installer is ~150 MB. The user data folder grows with your chat history (a few MB) and any cached models (Whisper STT ~75 MB, Tesseract OCR ~10 MB per language, Piper TTS ~70 MB per voice). Ollama models themselves live in Ollama's data directory, separate from PAiA.

Does PAiA work on Wayland Linux?

Mostly yes. Active-window detection works on GNOME (with the window-calls extension), KDE Plasma, and Hyprland. X11 is fully supported via xdotool.

Can I run my own LLM provider?

Yes — use the OpenAI-compatible provider with the base URL of any OpenAI-compatible server: LM Studio, vLLM, Together, OpenRouter, Groq, Cerebras, you name it.
